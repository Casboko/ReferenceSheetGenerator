# プロンプト設計（MVP）

- 目的: MVPではユーザ入力を極力そのまま渡し、安全と一貫性を担保。
- 想定読者: 実装/プロンプトデザイナ。

## 基本方針（MVP）
- ユーザ自由入力を許容、過度なテンプレート化は行わない
- 出力はTEXT+IMAGEを要求（textは任意表示）

## 既定値
- temperature: 0.4（例）
- candidateCount: 1（固定）

## 禁止/注意
- 肖像権/著作権を侵害しうる明示名指し
- 暴力/成人/差別的表現の強要

## 例（編集）
- 例: 「参照画像の人物に似せて、正面向きの上半身を明るい背景で。髪は短め、服はカジュアル。」

## 完成定義（DoD）
- api / ui と齟齬がない
- 安全上の注意が明文化されている

## 更新トリガー
- 出力品質要件の変化、モデル挙動の変化
